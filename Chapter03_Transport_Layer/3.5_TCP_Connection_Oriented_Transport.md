## ✅ 知识点1: 滑动窗口协议（GBN与SR）

**理论**

**GBN（Go-Back-N，回退N步）**
* **发送方**
  * 窗口尺寸为N，允许最多N个未确认分组
  * 收到确认后窗口滑动
  * 超时重传所有已发送未确认的分组
* **接收方**
  * 只接收按序到达的分组
  * 乱序分组直接丢弃
  * 累积确认（ACK n表示n及之前所有分组已收到）

**SR（Selective Repeat，选择重传）**
* **发送方**
  * 单独确认每个分组
  * 只重传丢失的分组
* **接收方**
  * 缓存乱序到达的分组
  * 单独确认每个正确接收的分组
  * 窗口滑动时按序交付

**对比GBN和SR**

| 特性 | GBN | SR |
|------|-----|-----|
| **窗口大小** | N | N |
| **接收方缓存** | 无 | 有 |
| **确认方式** | 累积确认 | 单独确认 |
| **重传策略** | 重传所有未确认 | 只重传丢失的 |
| **窗口最大尺寸** | ≤ 2^k - 1 | ≤ 2^(k-1) |

**窗口最大尺寸限制**
* GBN：窗口最大尺寸 ≤ 序号空间 - 1
* SR：窗口最大尺寸 ≤ 序号空间 / 2
* 原因：避免新旧分组序号重叠导致混淆

## ✅ 知识点2: TCP报文段结构

**理论**

**TCP报文段头部格式**

| 字段 | 长度 | 说明 |
|------|------|------|
| **源端口号** | 16比特 | 发送方端口号 |
| **目的端口号** | 16比特 | 接收方端口号 |
| **序号 (Sequence Number)** | 32比特 | 报文段首字节在字节流中的编号 |
| **确认号 (Acknowledgment Number)** | 32比特 | 期望收到的下一个字节序号（累积确认） |
| **首部长度** | 4比特 | TCP头部长度（以32比特字为单位） |
| **保留** | 6比特 | 保留未用 |
| **标志位 (URG, ACK, PSH, RST, SYN, FIN)** | 6比特 | 控制连接状态 |
| **接收窗口 (Receive Window)** | 16比特 | 流量控制用，接收方可用缓存大小 |
| **校验和 (Checksum)** | 16比特 | 差错检测 |
| **紧急数据指针 (Urgent Pointer)** | 16比特 | 紧急数据位置（通常不用） |
| **选项 (Options)** | 可变 | 如窗口扩展、时间戳等 |

**标志位说明**

| 标志 | 名称 | 功能 |
|------|------|------|
| **URG** | 紧急 | 紧急数据（通常不用） |
| **ACK** | 确认 | ACK字段合法 |
| **PSH** | 推送 | 马上推送数据（通常不用） |
| **RST** | 复位 | 连接复位 |
| **SYN** | 同步 | 建立连接 |
| **FIN** | 结束 | 关闭连接 |

<a id="id3"></a>
## ✅ 知识点3: TCP序号与确认号

**理论**

**序号 (Sequence Number)**
* **对字节计数（而不是报文段）**
* **报文段首字节在字节流中的编号**

**确认号 (Acknowledgment Number)**
* **期望从另一方收到的下一个字节的序号**
* **累积确认**：确认号n表示n-1及之前所有字节已正确收到

**示例**
```
发送方发送：Seq=92, 8 bytes of data
接收方回复：ACK=100  （期望收到第100字节）
```

**接收方处理乱序**
* TCP规范没有明确规定如何处理乱序报文段
* 实践中：接收方缓存乱序段，等待缺段到达后一起交付

<a id="id4"></a>
## ✅ 知识点4: TCP往返延时（RTT）与超时

**理论**

**EstimatedRTT计算**
* **指数加权移动平均**
```
EstimatedRTT = (1 - α) * EstimatedRTT + α * SampleRTT
```
* **推荐值：α = 0.125**
* **过去样本的影响呈指数衰减**

**超时时间设置**
```
TimeoutInterval = EstimatedRTT + 4 * DevRTT
```

**DevRTT（偏差）计算**
```
DevRTT = (1 - β) * DevRTT + β * |SampleRTT - EstimatedRTT|
```
* **推荐值：β = 0.25**

**关键原则**
* 超时周期往往太长 → 重传前延时太大
* 超时时间应略大于EstimatedRTT
* 避免过早超时（导致不必要的重传）

## ✅ 知识点5: TCP重传机制（快速重传）

**理论**

**产生TCP ACK的建议 [RFC 1122, RFC 2581]**

| 接收方事件 | TCP接收方动作 |
|-----------|--------------|
| 期望序号报文段按序到达，所有之前数据已确认 | 延迟ACK，等待最多500ms |
| 期望序号报文段按序到达，另一个按序报文段等待ACK | 立即发送单个累积ACK |
| 比期望序号大的报文段乱序到达 | 立即发送重复ACK，指明下一个期待字节的序号 |
| 能部分或完全填充间隔的报文段到达 | 立即发送ACK |

**快速重传 (Fast Retransmit)**
* **问题**：超时周期往往太长，重传丢失报文段前延时太长
* **检测方法**：通过重复的ACK检测报文段丢失
  * 发送方通常连续发送大量报文段
  * 如果某个报文段丢失，可能产生多个重复ACK
* **触发条件**：如果发送方收到同一数据的**3个冗余ACK**
* **动作**：重传最小序号的段（在定时器过时之前）
* **假设**：跟在被确认数据后面的数据丢失了

**快速重传算法**
```
event: ACK received, with ACK field value of y
if (y > SendBase) {
    SendBase = y
    if (there are currently not-yet-acknowledged segments)
        start timer
} else {
    increment count of dup ACKs received for y
    if (count of dup ACKs received for y == 3) {
        resend segment with sequence number y
    }
}
```

## ✅ 知识点6: TCP流量控制

**理论**

**流量控制目标**
* **防止发送方发送速率太快，淹没接收方**

**接收窗口 (RcvWindow)**
```
RcvWindow = RcvBuffer - [LastByteRcvd - LastByteRead]
```

| 变量 | 含义 |
|------|------|
| **RcvBuffer** | 接收缓存总大小 |
| **LastByteRcvd** | 最后收到的字节序号 |
| **LastByteRead** | 上层最后读取的字节序号 |

**工作原理**
* 接收方在ACK报文中通告`RcvWindow`
* 发送方限制未确认数据量 ≤ `RcvWindow`
* 当`RcvWindow = 0`时，发送方停止发送（探测机制继续探测窗口大小）

**假设**
* TCP接收方丢弃乱序的报文段

## ✅ 知识点7: TCP连接管理（3次握手/关闭）

**理论**

**为什么需要握手**
* 同意建立连接（每一方都知道对方愿意建立连接）
* 同意连接参数（如初始序号）

**2次握手的问题**
* **变化的延迟**：连接请求的段没有丢，但可能超时
* **丢失造成的重传**：如`req_conn(x)`丢失后重传
* **失败场景**
  ```
  客户端发送req_conn(x)
  服务器收到后进入ESTAB
  客户端未收到确认，重传req_conn(x)
  服务器为同一连接建立两个状态！
  ```

**TCP 3次握手**

| 步骤 | 客户端 | 方向 | 服务器 | 说明 |
|------|--------|------|--------|------|
| **1** | 选择初始序号x，发送SYN | → | LISTEN | SYNbit=1, Seq=x |
| **2** | | ← | 选择初始序号y，发送SYNACK | SYNbit=1, Seq=y, ACKbit=1, ACKnum=x+1 |
| **3** | 接收SYNACK，发送ACK | → | | ACKbit=1, ACKnum=y+1 |
| **结果** | ESTAB | | ESTAB | 双方建立连接 |

**状态变迁**
```
客户端：CLOSED → SYN_SENT → ESTABLISHED
服务器：CLOSED → LISTEN → SYN_RCVD → ESTABLISHED
```

**TCP关闭连接（4次挥手）**

| 步骤 | 动作 | 说明 |
|------|------|------|
| **1** | 客户端发送FIN | FINbit=1, seq=x，关闭发送端 |
| **2** | 服务器回复ACK | ACKbit=1, ACKnum=x+1 |
| **3** | 服务器发送FIN | FINbit=1, seq=y，关闭发送端 |
| **4** | 客户端回复ACK | ACKbit=1, ACKnum=y+1 |

**关闭状态变迁**
```
客户端：ESTAB → FIN_WAIT_1 → FIN_WAIT_2 → TIME_WAIT → CLOSED
服务器：ESTAB → CLOSE_WAIT → LAST_ACK → CLOSED
```

**TIME_WAIT状态**
* 等待2倍最大报文段生命周期(2*MSL)
* 确保最后一个ACK能到达服务器
* 防止已关闭连接的延迟报文干扰新连接

**同时关闭**
* 双方同时发送FIN
* 可以处理同时的FIN交换

## 🔑 核心要点总结
1. **GBN vs SR**：GBN累积确认、重传N个、接收方不缓存；SR单独确认、只重传丢失的、接收方缓存乱序
2. **TCP头部**：20字节基础头部，含序号、确认号、窗口、标志位
3. **序号/确认号**：对字节流编号，累积确认
4. **RTT估计**：指数加权移动平均，超时=EstimatedRTT+4*DevRTT
5. **快速重传**：3个重复ACK触发重传，不用等超时
6. **流量控制**：接收窗口限制发送方，防止淹没接收方
7. **3次握手**：SYN→SYNACK→ACK，防止失效连接请求
8. **4次挥手**：FIN→ACK→FIN→ACK，双方分别关闭

## 📌 考试速记版
* **GBN**：窗口N个，累积确认，超时重传N个，接收方不缓存乱序
* **SR**：窗口N个，单独确认，只重传丢失，接收方缓存乱序
* **TCP头部20字节**：源/目的端口(各16bit)、序号(32bit)、确认号(32bit)、窗口(16bit)、标志(6bit)
* **序号/确认号**：字节流编号，累积确认，ACK=n表示期望第n字节
* **RTT公式**：EstimatedRTT=(1-α)*EstimatedRTT+α*SampleRTT（α=0.125）
* **超时**：Timeout=EstimatedRTT+4*DevRTT
* **快速重传**：3个重复ACK立即重传
* **流量控制**：RcvWindow=RcvBuffer-(LastByteRcvd-LastByteRead)
* **3次握手**：SYN→SYNACK→ACK，防止失效请求
* **4次挥手**：FIN→ACK→FIN→ACK，TIME_WAIT等2MSL

## 🔍 GBN vs SR 对比表

| 特性 | GBN (Go-Back-N) | SR (Selective Repeat) |
|------|-----------------|----------------------|
| **窗口大小** | N | N |
| **发送方缓存** | 缓存N个 | 缓存N个 |
| **接收方缓存** | 无（只接受按序） | 缓存乱序分组 |
| **确认方式** | 累积ACK | 单独ACK |
| **重传数量** | 重传所有未确认 | 只重传丢失的 |
| **序号空间** | ≤ 2^k - 1 | ≤ 2^(k-1) |
| **复杂度** | 简单 | 较复杂 |
| **适用场景** | 低错误率信道 | 高错误率信道 |

## 🔍 TCP连接管理状态表

| 状态 | 含义 | 位置 |
|------|------|------|
| **CLOSED** | 关闭 | 初始/最终 |
| **LISTEN** | 监听连接请求 | 服务器 |
| **SYN_SENT** | 已发SYN，等待SYNACK | 客户端 |
| **SYN_RCVD** | 收到SYN，已发SYNACK | 服务器 |
| **ESTABLISHED** | 连接建立，数据传输 | 双方 |
| **FIN_WAIT_1** | 已发FIN，等待ACK | 主动关闭方 |
| **FIN_WAIT_2** | 收到ACK，等待对方FIN | 主动关闭方 |
| **CLOSE_WAIT** | 收到FIN，已发ACK，等待关闭 | 被动关闭方 |
| **LAST_ACK** | 已发FIN，等待最后ACK | 被动关闭方 |
| **TIME_WAIT** | 已发最后ACK，等待2MSL | 主动关闭方 |

**记忆口诀**：TCP段结构二十字节头，序号确认号字节流，GBN回退N累积确认，SR选择重传单独ACK，RTT指数加权估计，三个重复ACK快重传，窗口控制流量防淹没，三次握手四次挥手，SYN到ACK建连拆连

## ✅ 知识点1: 传输层目标与服务概述

**理论**

**传输层目标**
* **理解传输层的工作原理**
* **学习Internet的传输层协议**
  * 多路复用/解复用 (Multiplexing/Demultiplexing)
  * 可靠数据传输 (Reliable Data Transfer)
  * 流量控制 (Flow Control)
  * 拥塞控制 (Congestion Control)

**传输层核心原理**
* **为运行在不同主机上的应用进程提供逻辑通信 (Logical Communication)**
  * 传输协议运行在端系统 (End Systems)
  * 发送方：将应用层的报文 (Message) 分成报文段 (Segment)，然后传递给网络层
  * 接收方：将报文段重组成报文，然后传递给应用层

<a id="id2"></a>
## ✅ 知识点2: 传输服务和协议

**理论**

**传输层提供的服务**
* **为运行在不同主机上的应用进程提供逻辑通信**
* **传输协议运行在端系统**
  * 发送方：将应用层的报文分成报文段，然后传递给网络层
  * 接收方：将报文段重组成报文，然后传递给应用层

**可供选择的传输层协议**
* **Internet提供多个传输层协议可供选择**
  * **TCP (Transmission Control Protocol)**：面向连接的可靠传输
  * **UDP (User Datagram Protocol)**：无连接传输

<a id="id3"></a>
## ✅ 知识点3: 传输层 vs. 网络层

**理论**

**网络层服务 (Network Layer Services)**
* **主机之间的逻辑通信**
* **依赖于网络层的服务**
* **对网络层的服务进行增强**

**传输层服务 (Transport Layer Services)**
* **进程间的逻辑通信**
  * 依赖并增强网络层服务

**类比：东西两个家庭的通信**
* **主机 = 家庭** (Host = Family)
* **进程 = 小孩** (Process = Child)
* **应用层报文 = 信封中的信件** (Application Message = Letter in Envelope)
* **传输协议 = Ann和Bill** (Transport Protocol = Ann and Bill)
* **网络层协议 = 邮政服务** (Network Layer Protocol = Postal Service)
  * 家庭-家庭的邮包传输服务

**关键区分**
* **有些服务是可以加强的**：不可靠 → 可靠；安全
* **有些服务是不可以被加强的**：带宽、延迟
  * 带宽保证 (Bandwidth Guarantee)：无法加强
  * 延时保证 (Delay Guarantee)：无法加强

<a id="id4"></a>
## ✅ 知识点4: Internet传输层协议（TCP vs UDP）

**理论**

**TCP (Transmission Control Protocol) - 可靠的、保序的传输**
* **多路复用、解复用 (Multiplexing, Demultiplexing)**
* **拥塞控制 (Congestion Control)**
* **流量控制 (Flow Control)**
* **建立连接 (Connection Setup)**

**UDP (User Datagram Protocol) - 不可靠的、不保序的传输**
* **多路复用、解复用 (Multiplexing, Demultiplexing)**
* **没有为尽力而为 (Best-Effort) 的IP服务添加额外的服务**

**TCP和UDP都不提供的服务**
* **延时保证 (Delay Guarantee)**：不提供
* **带宽保证 (Bandwidth Guarantee)**：不提供

---

<a id="id5"></a>
## ✅ 知识点5: 多路复用与解复用概述

**理论**

**多路复用 (Multiplexing) - 在发送方主机**
* **从多个套接字 (Socket) 接收来自多个进程的报文**
* **根据套接字对应的IP地址和端口号等信息对报文段用头部加以封装**
  * 该头部信息用于以后的解复用

**多路解复用 (Demultiplexing) - 在接收方主机**
* **根据报文段的头部信息中的IP地址和端口号将接收到的报文段发给正确的套接字**
  * 从而交给对应的应用进程

**图示流程**
```
发送方:                                    接收方:
应用层 P1, P2, P3, P4                      应用层 P1, P2, P3, P4
    ↓                                         ↑
传输层 (多路复用) → 报文段 → 网络层 → 报文段 → 传输层 (多路解复用)
```

---

## 🔑 核心要点总结
1. **传输层核心功能**：为不同主机上的应用进程提供逻辑通信
2. **传输层 vs 网络层**：网络层提供主机间逻辑通信，传输层提供进程间逻辑通信
3. **TCP特点**：可靠、保序、拥塞控制、流量控制、建立连接
4. **UDP特点**：无连接、不可靠、不保序、低开销、无额外服务
5. **多路复用/解复用**：发送方将多个进程数据封装发送，接收方根据头部信息分发给正确进程
6. **都不提供的服务**：延时保证、带宽保证

## 📌 考试速记版
* **传输层核心**：进程间逻辑通信，端系统运行
* **TCP vs UDP**：
  * TCP：可靠、保序、有连接、拥塞控制、流量控制
  * UDP：无连接、不可靠、无拥塞控制、低开销
* **类比记忆**：主机=家庭，进程=小孩，传输协议=家长，网络协议=邮政
* **多路复用**：发送方封装（多个进程→一个通道）
* **多路解复用**：接收方分发（一个通道→多个进程）
* **无法加强的服务**：带宽、延迟（物理限制）

## 🔍 TCP vs UDP 服务对比表

| 特性 | TCP | UDP |
|------|-----|-----|
| **连接类型** | 面向连接 (Connection-oriented) | 无连接 (Connectionless) |
| **可靠性** | 可靠传输 (Reliable) | 不可靠传输 (Unreliable) |
| **保序性** | 保序 (In-order Delivery) | 不保序 (No Order Guarantee) |
| **拥塞控制** | 有 (Congestion Control) | 无 (No Congestion Control) |
| **流量控制** | 有 (Flow Control) | 无 (No Flow Control) |
| **多路复用/解复用** | 有 | 有 |
| **延时保证** | 不提供 | 不提供 |
| **带宽保证** | 不提供 | 不提供 |

## 🔍 传输层 vs 网络层对比表

| 特性 | 传输层 (Transport Layer) | 网络层 (Network Layer) |
|------|-------------------------|----------------------|
| **通信对象** | 进程间 (Process-to-Process) | 主机间 (Host-to-Host) |
| **逻辑通信** | 进程间的逻辑通信 | 主机间的逻辑通信 |
| **运行位置** | 端系统 (End Systems) | 路由器 + 端系统 |
| **服务增强** | 可增强（如不可靠→可靠） | 基础服务 |
| **无法加强** | 带宽、延迟 | - |

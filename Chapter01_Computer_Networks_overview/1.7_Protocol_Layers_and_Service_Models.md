<a id="id1"></a>
## ✅ 知识点1: 协议层次的概念

**理论**
* **问题**：如何组织和实现复杂的网络功能？
* **层次化方式实现复杂网络功能**：
  * 将网络复杂的功能分层功能明确的层次
  * 每一层实现了其中一个或一组功能
  * 功能中有其上层可以使用的功能：服务
* **层次间关系**：
  * 本层协议实体相互交互执行本层的协议动作
  * 目的是实现本层功能，通过接口为上层提供更好的服务
  * 在实现本层协议的时候，直接利用了下层所提供的服务
* **服务的定义**：
  * 借助下层服务实现的本层协议实体之间交互带来的新功能（上层可以利用的）
  * 加上更下层所提供的服务

**生活实例（航线系统）**
* 票务（购买/投诉）
* 行李（托运/认领）
* 登机口（登机/离机）
* 跑道（起飞/着陆）
* 按航线飞行

**注意点**
* 💡 **理解技巧**：分层就像公司组织架构，每层各司其职，向上层提供服务，依赖下层支持

<a id="id2"></a>
## ✅ 知识点2: 服务和服务访问点

**理论**
* **服务（Service）**：低层实体向上层实体提供它们之间的通信的能力
  * 服务用户（service user）
  * 服务提供者（service provider）
* **原语（primitive）**：上层使用下层服务的形式
  * 高层使用低层提供的服务
  * 低层向高层提供服务
  * 都是通过服务访问原语来进行交互的 — 形式
* **服务访问点SAP（Services Access Point）**：上层使用下层提供的服务通过层间的接口—地点
  * 例子：邮箱
  * 地址（address）：下层的一个实体支撑着上层的多个实体，SAP有标志不同上层实体的作用
  * 例子：传输层的SAP — 端口（port）

**注意点**
* 📋 **术语提醒**：
  * SAP（Service Access Point）：服务访问点
  * 端口（port）：传输层SAP

<a id="id3"></a>
## ✅ 知识点3: 服务的类型

**理论**
* **面向连接的服务（Connection-oriented Service）**：
  * 连接（Connection）：两个通信实体为进行通信而建立的一种结合
  * 通信过程：建立连接 → 通信 → 拆除连接
  * 例子：网络层的连接被称为虚电路
  * 适用范围：对于大的数据块要传输；不适合小的零星报文
  * 特点：保序
  * 服务类型：
    * 可靠的信息流 — 传送页面（可靠的获得，通过接收方的确认）
    * 可靠的字节流 — 远程登录
    * 不可靠的连接 — 数字化声音

* **无连接的服务（Connectionless Service）**：
  * 两个对等层实体在通信前不需要建立一个连接，不预留资源
  * 不需要通信双方都是活跃
  * 特点：不可靠、可能重复、可能失序
  * 例子：IP分组，数据包
  * 适用范围：适合传送零星数据
  * 服务类型：
    * 不可靠的数据报 — 电子方式的函件
    * 有确认的数据报 — 挂号信
    * 请求回答 — 信息查询

**注意点**
* 🔄 **对比记忆**：面向连接像打电话（先拨号、再通话、最后挂断），无连接像寄信（直接发，不保证到达）

<a id="id4"></a>
## ✅ 知识点4: 服务与协议的区别

**理论**
* **服务（Service）**：
  * 低层实体向上层实体提供它们之间的通信的能力
  * 是通过原语（primitive）来操作的
  * **垂直方向**
* **协议（Protocol）**：
  * 对等层实体（peer entity）之间在相互通信的过程中，需要遵循的规则的集合
  * **水平方向**
* **服务与协议的联系**：
  * 本层协议的实现要靠下层提供的服务来实现
  * 本层实体通过协议为上层提供更高级的服务

**注意点**
* 💡 **理解技巧**：服务是"能力"（上下层之间），协议是"规则"（对等实体之间）

<a id="id5"></a>
## ✅ 知识点5: Internet协议栈

**理论**
* **应用层（Application Layer）**：网络应用
  * 为人类用户或者其他应用进程提供网络应用服务
  * FTP, SMTP, HTTP, DNS
* **传输层（Transport Layer）**：主机之间的数据传输
  * 在网络层提供的端到端通信基础上，细分为进程到进程
  * 将不可靠的通信变成可靠地通信
  * TCP, UDP
* **网络层（Network Layer）**：为数据报从源到目的选择路由
  * 主机到主机之间的通信，端到端通信，不可靠
  * IP, 路由协议
* **链路层（Link Layer）**：相邻网络节点间的数据传输
  * 两个相邻节点的通信，点到点通信，可靠或不可靠
  * PPP, 802.11（WiFi）, Ethernet
* **物理层（Physical Layer）**：在线路上传送bit

**协议栈层次图**：
```
┌─────────────┐
│  应用层      │  ← HTTP, FTP, SMTP, DNS
├─────────────┤
│  传输层      │  ← TCP, UDP
├─────────────┤
│  网络层      │  ← IP, 路由协议
├─────────────┤
│  链路层      │  ← PPP, Ethernet, WiFi
├─────────────┤
│  物理层      │  ← 传输bit
└─────────────┘
```

**注意点**
* 📋 **术语提醒**：Internet协议栈只有5层，没有表示层和会话层

<a id="id6"></a>
## ✅ 知识点6: ISO/OSI参考模型

**理论**
* **7层模型**：
  * 应用层
  * **表示层**：允许应用解释传输的数据，如加密、压缩、机器相关的表示转换
  * **会话层**：数据交换的同步，检查点，恢复
  * 传输层
  * 网络层
  * 链路层
  * 物理层

* **与Internet协议栈的区别**：
  * 互联网协议栈没有表示层和会话层这两层
  * 这些服务如果被应用需要，必须由应用自己实现

**OSI vs Internet协议栈对比**：
```
OSI 7层:          Internet 5层:
┌─────────┐       ┌─────────┐
│ 应用层   │       │ 应用层   │
├─────────┤       ├─────────┤
│ 表示层   │ ←×    │         │
├─────────┤       │         │
│ 会话层   │ ←×    │         │
├─────────┤       ├─────────┤
│ 传输层   │       │ 传输层   │
├─────────┤       ├─────────┤
│ 网络层   │       │ 网络层   │
├─────────┤       ├─────────┤
│ 链路层   │       │ 链路层   │
├─────────┤       ├─────────┤
│ 物理层   │       │ 物理层   │
└─────────┘       └─────────┘
```

**注意点**
* 💡 **记忆口诀**：OSI七层 — "应表会传网链物"（All People Seem To Need Data Processing）

<a id="id7"></a>
## ✅ 知识点7: 封装和解封装

**理论**
* **封装（Encapsulation）过程**：
  * 应用层报文 M
  * 传输层：添加传输层首部 Ht → 报文段 Ht|M
  * 网络层：添加网络层首部 Hn → 数据报 Hn|Ht|M
  * 链路层：添加链路层首部 Hl → 帧 Hl|Hn|Ht|M
  * 物理层：传输bit

* **解封装（Decapsulation）过程**：
  * 目标主机从下往上逐层去掉首部
  * 每层提取本层需要的信息，将剩余部分交给上层

* **各层次的协议数据单元**：
  * 应用层：报文（message）
  * 传输层：报文段（segment），TCP段，UDP数据报
  * 网络层：分组packet（如果无连接方式：数据报datagram）
  * 数据链路层：帧（frame）
  * 物理层：位（bit）

**封装过程图示**：
```
源端:                          目标端:
M          应用层                M
↑↓
Ht|M       传输层                Ht|M
↑↓
Hn|Ht|M    网络层                Hn|Ht|M
↑↓
Hl|Hn|Ht|M 链路层                Hl|Hn|Ht|M
↑↓
物理传输 ───────────────────────→
```

**注意点**
* 📋 **术语提醒**：
  * **封装**：添加首部信息
  * **解封装**：去除首部信息
  * 每层的PDU（Protocol Data Unit）有不同的名称

## 🔑 核心要点总结

1. **协议层次**：将复杂网络功能分层实现，每层利用下层服务，为上层提供服务
2. **服务类型**：
   * 面向连接：先建立连接再通信，保序（如TCP）
   * 无连接：直接发送，不保证到达（如UDP）
3. **服务 vs 协议**：
   * 服务是垂直的（上下层之间）
   * 协议是水平的（对等实体之间）
4. **Internet协议栈（5层）**：应用层、传输层、网络层、链路层、物理层
5. **OSI参考模型（7层）**：多了表示层和会话层，Internet协议栈中没有
6. **封装过程**：每层添加首部，形成不同PDU（报文、报文段、数据报、帧、bit）

## 📌 考试速记版

* **Internet 5层模型**：

| 层次 | 功能 | 代表协议 | PDU |
|------|------|----------|-----|
| 应用层 | 网络应用 | HTTP, FTP, SMTP, DNS | 报文 |
| 传输层 | 进程到进程 | TCP, UDP | 报文段 |
| 网络层 | 路由选择 | IP, 路由协议 | 数据报 |
| 链路层 | 点到点传输 | Ethernet, WiFi, PPP | 帧 |
| 物理层 | 传输bit | - | bit |

* **OSI 7层口诀**：应表会传网链物（All People Seem To Need Data Processing）

* **服务 vs 协议**：
  * 服务 = 垂直 + 能力
  * 协议 = 水平 + 规则

* **封装过程**：
  * 从上往下：加首部（应用→传输→网络→链路）
  * 从下往上：去首部（物理→链路→网络→传输→应用）

**记忆口诀**：
> "五层模型要记牢，应传网链物层层包；
> 水平协议垂直服，封装解封首尾添。"

## ✅ 知识点1: BGP概述——"互联网的胶水"

**理论**
- **BGP (Border Gateway Protocol)**：自治区域间路由协议"事实上的"标准
  - **"将互联网各个AS粘在一起的胶水"**
- BGP提供给每个AS以以下方法：
  - **eBGP**：从相邻的ASes那里获得子网可达信息
  - **iBGP**：将获得的子网可达信息传遍到AS内部的所有路由器
  - 根据子网可达信息和策略来决定到达子网的"好"路径
- 允许子网向互联网其他网络通告"我在这里"
- **基于距离矢量算法（路径矢量）**
  - 不仅仅是距离矢量，还包括到达各个目标网络的**详细路径（AS序号的列表）**
  - 能够避免简单DV算法的路由环路问题

> 💡 **理解技巧**：BGP就像"国际外交协议"——每个AS是一个国家，BGP负责让各国互通有无，同时各国可以按自己的"外交政策"选择路径。

> 📋 **术语提醒**：`BGP(Border Gateway Protocol)` / `路径矢量(Path Vector)` / `eBGP(External BGP)` / `iBGP(Internal BGP)`

<a id="id2"></a>
## ✅ 知识点2: eBGP与iBGP连接

**理论**
- **eBGP connectivity**：不同AS之间的BGP连接
- **iBGP connectivity**：同一AS内部的BGP连接
- **网关路由器**同时运行eBGP和iBGP协议

![eBGP与iBGP连接](images/page_19.png)

- eBGP用于AS之间交换路由信息
- iBGP用于AS内部传播从eBGP获得的路由信息

> ⚠️ **关键区分**：eBGP是"跨国贸易"，iBGP是"国内流通"。网关路由器是"海关"，同时处理进出口和国内分发。

> 📋 **术语提醒**：`网关路由器(Gateway Router)` / `对等体(Peer)`

<a id="id3"></a>
## ✅ 知识点3: BGP基础——路径通告语义

**理论**
- **BGP会话**：2个BGP路由器("peers")在一个**半永久的TCP连接**上交换BGP报文
  - 通告向不同目标子网前缀的"路径"
  - BGP是一个**"路径矢量"协议**
- **路径通告示例**：
  - 当AS3网关路由器3a向AS2的网关路由器2c通告路径：AS3,X
    - 3a参与AS内路由运算，知道本AS所有子网X信息
    - **语义上**：AS3向AS2承诺，它可以向子网X转发数据报
    - 3a是2c关于X的**下一跳（next hop）**

![BGP路径通告](images/page_19.png)

> 💡 **理解技巧**：BGP的"路径"不是"怎么走"，而是"经过哪些AS"——AS-PATH就像护照上的签证章序列。

<a id="id4"></a>
## ✅ 知识点4: 路径属性与BGP路由

**理论**
- 当通告一个子网前缀时，通告包括BGP属性
  - **prefix + attributes = "route"**
- **2个重要的属性**：
  1. **AS-PATH**：前缀的通告所经过的AS列表（如：AS67 AS17）
     - **检测环路**：如果发现自己的ASN已经在路径中，就拒绝该路径
     - **多路径选择**：用于选择较短或较优路径
     - 在向其它AS转发时，需要将自己的AS号加在路径上
  2. **NEXT-HOP**：从当前AS到下一跳AS有多个链路时，告诉对方通过哪个接口转发
  3. **其它属性**：路由偏好指标、如何被插入的属性
- **基于策略的路由**：
  - 当一个网关路由器接收到了一个路由通告，使用**输入策略**来接受或过滤（accept/decline）
    - 过滤原因例1：不想经过某个AS，转发某些前缀的分组
    - 过滤原因例2：已经有了一条往某前缀的偏好路径
  - 策略也决定了是否向它别的邻居通告收到的这个路由信息

> ⚠️ **核心机制**：AS-PATH是BGP防环路的关键——每个AS在转发时把自己的ASN加到路径前面，形成"签名链"。

> 💡 **理解技巧**：NEXT-HOP就像"请从这个门进"，AS-PATH就像"我经过了这些检查站"。

> 📋 **术语提醒**：`AS-PATH` / `NEXT-HOP` / `前缀(Prefix)` / `策略路由(Policy-based Routing)`

<a id="id5"></a>
## ✅ 知识点5: BGP路径通告过程

**理论**
- **路径通告流程**：
  1. 路由器AS2.2c从AS3.3a接收到AS3,X路由通告（通过**eBGP**）
  2. 基于AS2的输入策略，AS2.2c决定接收AS3,X的通告，而且通过**iBGP**向AS2的所有路由器进行通告
  3. 基于AS2的策略，AS2路由器2a通过**eBGP**向AS1.1c路由器通告AS2,AS3,X路由信息
     - **路径上加上了AS2自己作为AS序列的一跳**

![BGP路径通告过程](images/page_20.png)

- **多路径选择**：
  - AS1网关路由器1c从2a学习到路径：AS2,AS3,X
  - AS1网关路由器1c从3a处学习到路径AS3,X
  - 基于策略，AS1路由器1c选择了路径：AS3,X（更短，少经过一个AS）
  - 通过**iBGP**告诉所有AS1内部的路由器

![BGP多路径选择](images/page_20.png)

> 💡 **理解技巧**：BGP路径选择不只是"最短"，更是"最符合策略"——可能选择绕远但避开某个AS的路径。

<a id="id6"></a>
## ✅ 知识点6: BGP报文类型

**理论**
- BGP使用**TCP协议**交换报文（端口179）
- **BGP报文类型**：
  1. **OPEN**：打开TCP连接，认证发送方
  2. **UPDATE**：通告新路径（或者撤销原路径）
  3. **KEEPALIVE**：在没有更新时保持连接，也用于对OPEN请求确认
  4. **NOTIFICATION**：报告以前消息的错误，也用来关闭连接

| 报文类型 | 作用 |
|:---|:---|
| OPEN | 建立TCP连接，认证身份 |
| UPDATE | 通告/撤销路由 |
| KEEPALIVE | 保活，确认连接 |
| NOTIFICATION | 报告错误，关闭连接 |

> 📋 **术语提醒**：`UPDATE报文` / `KEEPALIVE报文` / `NOTIFICATION报文`

## ✅ 知识点7: BGP、OSPF、转发表的综合例子

**理论**
- **转发表是如何设置的**：
  - 1a, 1b, 1c从1c那里通过**iBGP**学习到了子网X，"到往子网X的要通过1c"
  - 1d：通过**OSPF**内部网关协议，为了到达1c，必须通过本地接口1
- **路由表项是由AS内和AS间的路由共同决定的**

![BGP/OSPF/转发表综合](images/page_21.png)

**综合决策过程**：
1. **AS间路由（BGP）**：决定"去子网X应该走哪个网关路由器"（如1c）
2. **AS内路由（OSPF）**：决定"去1c应该走哪个本地接口"（如接口1）
3. **转发表**：综合以上两者，生成 `dest=X, interface=1` 的表项

> 💡 **理解技巧**：BGP负责"跨AS导航"（选网关），OSPF负责"AS内导航"（选接口），两者配合生成最终转发表。

> 🔄 **知识关联**：这体现了"层次路由"的本质——BGP解决"国家间"问题，OSPF解决"国内"问题。

## 🔑 核心要点总结

1. **BGP是AS间的"外交协议"**：负责在自治系统之间交换可达性信息，"把互联网粘在一起"。
2. **eBGP vs iBGP**：eBGP用于AS之间交换，iBGP用于AS内部传播eBGP获得的信息。
3. **路径矢量 = DV + 路径信息**：AS-PATH记录经过的AS序列，既能选路又能防环路。
4. **策略路由是BGP的核心**：AS可以根据商业策略（如避开竞争对手AS）来选择或过滤路径。
5. **转发表由两层路由共同决定**：BGP选网关（AS间），OSPF选接口（AS内）。

## 📌 考试速记版

- **关键概念**：
  - BGP = Border Gateway Protocol，路径矢量协议
  - AS-PATH = 经过的AS列表（防环路+选路）
  - NEXT-HOP = 进入下一AS的接口
  - eBGP = AS间，iBGP = AS内
- **重要对比**：

| | OSPF (intra-AS) | BGP (inter-AS) |
|:---|:---|:---|
| 范围 | AS内部 | AS之间 |
| 算法 | LS | 路径矢量 |
| 传输 | IP直接 | TCP (端口179) |
| 目标 | 找到最短路径 | 找到符合策略的路径 |
| 代价 | 链路代价 | AS-PATH长度+策略 |

- **常见考点**：
  - BGP为什么用TCP而不是UDP？→ 需要可靠传输，半永久连接
  - AS-PATH如何防环路？→ 发现自身ASN已在路径中则拒绝
  - 为什么BGP叫"路径矢量"不叫"距离矢量"？→ 不仅传距离，还传完整AS路径
  - 转发表怎么由BGP和OSPF共同决定？→ BGP选网关，OSPF选接口

**记忆口诀**：
> "BGP粘AS，eBGP对外iBGP对内传；AS-PATH像护照章，策略过滤不简单；TCP半永久连，UPDATE通告KEEPALIVE保活关。"

## ✅ 知识点1: 网络层服务

**服务概览**
- 传输层在**发送主机和接收主机对**之间传送**段(segment)**
- 在**发送端**将段向下传递给网络层<b>封装到数据报(datagram)</b>中并发送
- 在**接收端**，将段**上交给传输层实体**并讲数据包解封装
- 网络层协议存在于每一个主机和路由器
- 路由器检查每一个经过它的IP数据报的头部


**注意点**
- ⚠️ **关键角色**：路由器是网络层的核心设备，每个经过的数据报都要检查头部
- 💡 **理解技巧**：可以把路由器想象成邮局分拣员，每个包裹（数据报）都要看地址标签（IP头部）决定往哪送

<a id="id2"></a>
## ✅ 知识点2: 关键功能：转发与路由


网络层有两个关键功能：

- **转发(Forwarding)**：将分组从路由器的**输入接口**转发到**合适的输出接口**
  - 类比：通过**单个路口**的过程
- **路由(Routing)**：使用<b>路由算法(routing algorithm)</b>来决定分组从发送主机到目标接收主机的路径
  - 包含：**路由选择算法**、**路由选择协议**
  - 类比：从源到目的的**路径规划过程**

**注意点**
- ⚠️ **核心区别**：**转发是本地动作**（单个路由器内），**路由是全局决策**（端到端路径）
- 🔄 **知识关联**：转发对应**数据平面**，路由对应**控制平面**
- 📋 **术语提醒**：Forwarding ≠ Routing，面试常考区别

<a id="id3"></a>
## ✅ 知识点3: 数据平面

**理论**
- **数据平面(Data Plane)**是**本地**功能，属于**每个路由器**的功能
- 决定从路由器**输入端口**到达的分组如何转发到**输出端口**
- **转发功能**有两种实现方式：
  - **传统方式**：基于**目标地址** + **转发表**
  - **SDN(software defined network)方式**：基于**多个字段** + **流表**
    - 并且可以进行多种操作：如字段修改，泛洪，转发，阻挡等等
    - 操作由流表决定，流表由特殊网络操作系统生成

**注意点**
- ⚠️ **本质特征**：数据平面是**本地、逐跳、实时**的分组处理
- 💡 **理解技巧**：数据平面就像高速公路收费站的实时放行决策，只看当前车辆信息决定开哪个闸口
- 🔄 **知识关联**：传统方式查**转发表**，SDN方式查**流表**

---

<a id="id4"></a>
## ✅ 知识点4: 控制平面

**理论**
- **控制平面**是**网络范围内的逻辑**
- 决定数据报如何在路由器之间**路由**，决定从源到目标主机之间的**端到端路径(end-to-end path)**
- **2个控制平面方法**：
  1. **传统路由算法(traditional routing algorithms)**：在**路由器中被实现**
  2. **软件定义网络(Software-Defined Networking, SDN)**：在**远程的服务器(remote servers)中实现**

**注意点**
- ⚠️ **核心区别**：控制平面负责"全局路径规划"，数据平面负责"本地转发执行"
- 💡 **理解技巧**：控制平面像城市交通指挥中心（决定哪条路通畅），数据平面像路口红绿灯（执行具体放行）
- 🔄 **知识关联**：传统方式 = 分布式决策（每个路由器自己算），SDN = 集中式决策（远程控制器算）

---

<a id="id5"></a>
## ✅ 知识点5: 传统方式：每一路由器控制平面

**理论**
- **传统方式(Traditional Approach)**采用**每一路由器(Per-router)**控制平面
- 在**每一个路由器**中存在**单独的路由器算法元件**
- 这些算法元件在**控制平面**进行交互（路由器之间互相交换路由信息）

**注意点**
- ⚠️ **架构特征**：控制平面功能是**分布式**的，每个路由器独立运行路由算法
- 💡 **理解技巧**：就像每个路口都有自己的导航APP，互相打听路况后各自决定怎么指路
- 🔄 **知识关联**：这是互联网传统路由协议（如OSPF、BGP）的工作方式

<a id="id6"></a>
## ✅ 知识点6: 路由和转发的相互作用

**理论**
- **控制平面**：**路由算法(routing algorithm)**决定**端到端路径(end-to-end path)**
- **数据平面**：**IP协议**根据**转发表(forwarding table)**决定了IP数据报在此路由器上的**局部转发(local forwarding)**
- 工作流程：
  1. 路由选择算法计算路径 → 生成**本地转发表(local forwarding table)**
  2. 转发表包含：**头部值(header value)** → **输出链路(output link)** 的映射
  3. 到达分组的头部值匹配转发表 → 确定输出接口

**注意点**
- ⚠️ **关键机制**：**控制平面生成转发表，数据平面使用转发表**
- 💡 **理解技巧**：路由算法像地图规划师（画好路线图做成转发表），IP转发像司机（看路牌/导航表开车）
- 🔄 **知识关联**：转发表是控制平面和数据平面的**接口(interface)**

<a id="id7"></a>
## ✅ 知识点7: SDN方式：逻辑集中的控制平面

**理论**
- **SDN方式(SDN Approach)**采用**逻辑集中的控制平面(logically centralized control plane)**
- 一个**不同的（通常是远程的）控制器(remote controller)**与**本地控制代理(Local Control Agents, CAs)**交互
- 架构组成：
  - **Remote Controller**：远程控制器，集中计算流表
  - **CA (Control Agent)**：每个路由器/分组交换机上的本地控制代理
  - **分组交换机(Packet Switches)**：只负责按流表转发数据

**注意点**
- ⚠️ **"逻辑集中" ≠ "物理集中"**：控制器可以是分布式部署的，但**逻辑上**是一个统一实体
- 💡 **理解技巧**：像连锁店的中央厨房（远程控制器）给各分店（路由器）发菜谱（流表），分店只负责按菜谱做菜（转发）
- 🔄 **知识关联**：SDN将**控制平面**从路由器中抽离，放到远程服务器，路由器只保留**数据平面**

<a id="id8"></a>
## ✅ 知识点8: 网络服务模型

**理论**

**问题定义**：从发送方主机到接收方主机传输数据报的"通道"，网络提供什么样的**服务模型(service model)**？

**对于单个数据报的服务**：
- **可靠传送(reliable delivery)**
- **延迟保证(delay guarantee)**，如：少于40ms的延迟

**对于数据报流的服务**：
- **保序数据报传送(in-order delivery)**
- **保证流的最小带宽(guaranteed minimum bandwidth)**
- **分组之间的延迟差(delay difference between packets)**

**网络层服务模型对比表**：

| 网络架构 | 服务模型 | 带宽 | 丢失 | 保序 | 延迟 | 拥塞反馈 |
|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **Internet** | best effort (尽力而为) | none | no | no | no | no (inferred via loss) |
| **ATM** | CBR (恒定速率) | constant rate | yes | yes | yes | no congestion |
| **ATM** | VBR (变化速率) | guaranteed rate | yes | yes | yes | no congestion |
| **ATM** | ABR (可用比特率) | guaranteed minimum | no | yes | no | yes |
| **ATM** | UBR (不指名比特率) | none | no | yes | no | no |

**注意点**
- ⚠️ **Internet的best effort**：**什么都不保证**，无带宽保证、无丢失保证、不保序、无延迟保证、无拥塞反馈（只能通过丢包推断）
- 💡 **理解技巧**：Internet像普通邮政——不保证时效、不保证不丢件、不保证按顺序到，但免费且普适；ATM像快递服务——分级付费，不同等级承诺不同服务质量
- 📋 **术语提醒**：best effort = 尽力而为，CBR = Constant Bit Rate，VBR = Variable Bit Rate，ABR = Available Bit Rate，UBR = Unspecified Bit Rate

<a id="id9"></a>
## ✅ 知识点9: 连接建立

**理论**
- 在某些网络架构中，连接建立是**第三个重要的功能**
  - 例如：**ATM**、**frame relay (帧中继)**、**X.25**
- 在分组传输之前，在两个主机之间，在通过一些路由器所构成的路径上**建立一个网络层连接**
  - **涉及到路由器**
- **网络层和传输层连接服务区别**：
  - **网络层**：在**2个主机**之间，涉及到路径上的**一些路由器**
  - **传输层**：在**2个进程**之间，很可能只体现在**端系统上**（如**TCP连接**）

**注意点**
- ⚠️ **重要区别**：网络层连接涉及**路由器参与建链**，传输层连接（如TCP）通常只在**端系统**上实现
- 💡 **理解技巧**：网络层连接像打电话时运营商帮你建立专线（中间交换机参与），传输层连接像两人约定暗号（只有通话双方知道）
- 🔄 **知识关联**：Internet的网络层是**无连接(connectionless)**的，不需要连接建立；ATM等是**面向连接(connection-oriented)**的

---

## 🔑 核心要点总结

1. **转发 vs 路由**：转发是本地路由器内的分组导向动作（数据平面），路由是端到端路径的全局决策（控制平面）
2. **两种控制平面架构**：传统方式（Per-router，分布式路由算法） vs SDN方式（逻辑集中，远程控制器+本地CA）
3. **Internet服务模型**：best effort，不保证带宽、丢失、保序、延迟和拥塞反馈
4. **网络层连接建立**：与传输层连接不同，网络层连接需要路径上的路由器参与（如ATM），而TCP连接仅在端系统上实现

## 📌 考试速记版

- **转发(Forwarding)** = 本地动作，查表决定输出接口
- **路由(Routing)** = 全局决策，计算端到端路径
- **数据平面** = 本地转发（传统查转发表 / SDN查流表）
- **控制平面** = 路径规划（传统分布式 / SDN集中式）
- **Internet best effort**：无带宽、无丢包保证、不保序、无延迟保证、无拥塞反馈
- **ATM四种服务**：CBR（恒定）、VBR（变化）、ABR（可用最小）、UBR（不指名）
- **网络层连接**涉及路由器，**传输层连接**只在端系统

**记忆口诀**：
> 转发本地查表快，路由全局算路径；  
> 数据平面管转发，控制平面管规划；  
> 传统分布各自算，SDN集中远程控；  
> Internet尽力传，ATM分级有保障。

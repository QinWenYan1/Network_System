<a id="id1"></a>
## ✅ 知识点1: FTP概述

**理论**
* **FTP**：**文件传输协议(File Transfer Protocol)**
* **功能**：向远程主机上传输文件或从远程主机接收文件
* **客户/服务器模式**
  * **客户端**：发起传输的一方
  * **服务器**：远程主机
* **标准**：RFC 959
* **端口**：FTP服务器使用**端口号21**

**注意点**
* 📋 **术语**：FTP (File Transfer Protocol) - 文件传输协议、RFC 959 - FTP标准文档

<a id="id2"></a>
## ✅ 知识点2: FTP的工作模式

**理论**
* **用户接口**：用户通过FTP客户端与服务器交互
* **本地文件系统**：客户端维护本地文件
* **远程文件系统**：服务器维护远程文件

```
用户/主机          FTP客户端                FTP服务器
   |                  |                        |
   |                  |    TCP控制连接(端口21)  |
   |----------------->|<---------------------->|
   |   本地文件系统   |       远程文件系统      |
```

<a id="id3"></a>
## ✅ 知识点3: 控制连接与数据连接分开

**理论**
* **控制连接 (TCP Control Connection)**
  * **服务器端口**：21
  * FTP客户端与FTP服务器通过端口21联系
  * 使用TCP作为传输协议
  * 客户端通过控制连接获得身份确认
  * 客户端通过控制连接发送命令浏览远程目录

* **数据连接 (TCP Data Connection)**
  * **服务器端口**：20
  * 收到一个文件传输命令时，服务器打开一个到客户端的数据连接
  * 一个文件传输完成后，服务器关闭连接

* **带外传输 ("Out of Band")**
  * 控制连接：带外传送
  * 数据连接：单独建立

**架构图**
```
                    TCP control connection
                   server port 21
         FTP                                FTP
       client                              server
          \                                  /
           \      TCP data connection       /
            \     server port 20          /
             \___________________________/
```

**注意点**
* ⚠️ **关键区别**：FTP使用**两个独立的TCP连接**，HTTP使用一个TCP连接
* 📋 **术语**：Control Connection - 控制连接、Data Connection - 数据连接、Out of Band - 带外传输

<a id="id4"></a>
## ✅ 知识点4: FTP的状态维护

**理论**
* **FTP服务器维护用户的状态信息**
  * 当前路径
  * 用户账户
  * 控制连接对应
* **有状态协议**
  * 服务器需要维护用户的状态信息
  * 与HTTP的无状态特性不同

**注意点**
* ⚠️ **对比**：FTP是**有状态协议**，HTTP是**无状态协议**

<a id="id5"></a>
## ✅ 知识点5: FTP命令与响应

**理论**

**传输方式**
* 在控制连接上以**ASCII文本**方式传送

**命令样例**

| 命令 | 说明 |
|------|------|
| `USER username` | 申明用户名 |
| `PASS password` | 输入口令 |
| `LIST` | 请服务器返回远程主机当前目录的文件列表(gets) |
| `RETR filename` | 从远程主机的当前目录检索文件(gets) |
| `STOR filename` | 向远程主机的当前目录存放文件(puts) |

**返回码样例**

| 返回码 | 说明 |
|--------|------|
| `331 Username OK, password required` | 用户名正确，需要密码 |
| `125 data connection already open; transfer starting` | 数据连接已打开，开始传输 |
| `425 Can't open data connection` | 无法打开数据连接 |
| `452 Error writing file` | 写文件错误 |

**状态码和状态信息**
* 同HTTP的状态码机制

**注意点**
* 📋 **术语**：USER - 用户名、PASS - 密码、LIST - 列表、RETR - 检索、STOR - 存储

---

## 🔑 核心要点总结
1. **FTP使用两个TCP连接**：控制连接(端口21) + 数据连接(端口20)
2. **有状态协议**：服务器维护用户状态（当前路径、用户账户）
3. **带外传输**：控制信息通过单独的控制连接传输
4. **命令格式**：ASCII文本，类似HTTP的命令-响应模式

## 📌 考试速记版
* **FTP双连接**：控制(21) + 数据(20)
* **有状态**：服务器记住用户状态
* **vs HTTP**：HTTP单连接、无状态；FTP双连接、有状态
* **常用命令**：USER/PASS(认证)、LIST(列目录)、RETR(下载)、STOR(上传)

**记忆口诀**：FTP二一端口控，二零端口传数据，有状态来带外走，命令响应ASCII流
